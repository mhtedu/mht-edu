# 棉花糖教育平台 - 数据库导入指南

## 问题说明

宝塔面板的 MySQL root 密码包含特殊字符时，在命令行中直接使用会报错：
```bash
# ❌ 错误方式（密码中有特殊字符会报错）
mysql -u root -p'abc#123$%' mht_edu < mht_edu_all_in_one.sql
# 报错：bash: !@#$: event not found
```

## 解决方案

### 方式一：phpMyAdmin 导入（推荐 ✅）

**步骤：**
1. 登录宝塔面板
2. 点击左侧菜单【数据库】
3. 点击数据库列表右侧的【管理】按钮，进入 phpMyAdmin
4. 在 phpMyAdmin 左侧选择 `mht_edu` 数据库（如果没有，先创建）
5. 点击顶部【导入】选项卡
6. 点击【选择文件】，上传 `mht_edu_all_in_one.sql`
7. 点击底部【执行】按钮
8. 看到 "数据库初始化完成！" 即表示成功

---

### 方式二：宝塔面板数据库管理

**步骤：**
1. 登录宝塔面板
2. 点击左侧菜单【数据库】
3. 点击【添加数据库】，创建数据库 `mht_edu`
4. 点击数据库右侧的【导入】按钮
5. 选择 `mht_edu_all_in_one.sql` 文件
6. 点击【导入】

---

### 方式三：命令行交互式输入密码

```bash
# ✅ 正确方式：使用 -p 参数但不跟密码，系统会提示输入
mysql -u root -p < /www/wwwroot/mht-edu/deploy_package/server/database/mht_edu_all_in_one.sql

# 系统提示：Enter password:
# 此时手动输入密码（不会显示），然后按回车
```

---

### 方式四：创建 MySQL 配置文件（避免每次输入密码）

```bash
# 1. 创建配置文件
cat > ~/.my.cnf << 'EOF'
[client]
user=root
password=你的MySQL密码
EOF

# 2. 设置权限（重要！）
chmod 600 ~/.my.cnf

# 3. 导入数据库（不需要再输入密码）
mysql < /www/wwwroot/mht-edu/deploy_package/server/database/mht_edu_all_in_one.sql

# 4. 导入成功后可删除配置文件（安全考虑）
rm ~/.my.cnf
```

---

### 方式五：使用 MySQL 配置编辑器（加密存储密码）

```bash
# 1. 创建加密的登录路径（只需执行一次）
mysql_config_editor set --login-path=local --host=localhost --user=root --password
# 系统会提示输入密码

# 2. 导入数据库（使用加密存储的密码）
mysql --login-path=local < /www/wwwroot/mht-edu/deploy_package/server/database/mht_edu_all_in_one.sql

# 3. 查看已保存的登录路径
mysql_config_editor print --all
```

## 导入后的配置

### 1. 修改管理员密码

登录管理后台后，**立即修改默认密码**：
- 默认账号：`admin`
- 默认密码：`admin123`

### 2. 配置站点信息

在管理后台【系统设置】中配置：
- 微信小程序 AppID 和 AppSecret
- 微信支付商户号和密钥
- 地图 API Key（腾讯地图）
- 短信服务 AccessKey（阿里云）

### 3. 配置服务器

修改 NestJS 后端配置文件：
```bash
vi /www/wwwroot/mht-edu/server/.env

# 数据库配置（如果与默认不同）
DB_HOST=localhost
DB_PORT=3306
DB_USERNAME=root
DB_PASSWORD=你的MySQL密码
DB_DATABASE=mht_edu
```

## 常见问题

### Q1: 导入时提示 "Unknown database 'mht_edu'"
**A:** 合并的SQL文件已包含创建数据库语句，直接导入即可。如果仍报错，请先手动创建数据库：
```sql
CREATE DATABASE mht_edu DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

### Q2: 导入时提示 "Access denied for user 'root'"
**A:** 密码错误，请在宝塔面板【数据库】页面查看正确的 root 密码。

### Q3: 导入后前端无法访问
**A:** 检查后端配置文件 `.env` 中的数据库连接信息是否正确。

### Q4: 如何完全重置数据库？
**A:** 合并的SQL文件已包含删除所有表的语句，重新导入即可完全重置。
