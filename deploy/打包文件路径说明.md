# 打包文件路径说明

## 📦 打包文件位置

项目根目录：`/workspace/projects/`

### 前端打包文件
```
/workspace/projects/dist-web/
├── index.html          # 入口HTML文件
├── css/                # 样式文件
├── js/                 # JavaScript文件
└── static/             # 静态资源（图片等）

大小：约 1.2MB
```

### 后端编译文件
```
/workspace/projects/server/dist/
├── main.js             # 入口文件（PM2启动这个）
├── app.module.js       # 应用模块
├── modules/            # 业务模块
└── ...

大小：约 1.8MB
```

---

## 🔌 端口配置（使用3002端口）

### 方法一：修改环境变量

在 `.env.production` 中添加：
```bash
PORT=3002
```

### 方法二：PM2启动时指定端口

```bash
pm2 start dist/main.js --name mht-edu-api -- -p 3002
```

### 方法三：修改Nginx反向代理

Nginx配置中的 `proxy_pass` 改为：
```nginx
location /api/ {
    proxy_pass http://127.0.0.1:3002;  # 改为3002
    # ...其他配置
}
```

## 📋 部署需要上传的文件

### 上传到服务器的文件

从开发环境复制到服务器：

```bash
# 前端文件（上传整个文件夹）
/workspace/projects/dist-web/  →  /www/wwwroot/mht-edu/dist-web/

# 后端文件（上传整个文件夹）
/workspace/projects/server/  →  /www/wwwroot/mht-edu/server/
# 注意：server目录包含：
#   - dist/          编译后的代码（必须）
#   - package.json   依赖配置（必须）
#   - node_modules/  依赖包（可选，可在服务器安装）

# 部署配置
/workspace/projects/deploy/  →  /www/wwwroot/mht-edu/deploy/

# 数据库SQL文件
/workspace/projects/server/database/  →  /www/wwwroot/mht-edu/database/
```

## ⚡ 快速部署命令（使用3002端口）

```bash
# 1. 进入后端目录
cd /www/wwwroot/mht-edu/server

# 2. 安装依赖（如果没有node_modules）
npm install --production

# 3. 创建环境配置
cat > .env.production << 'EOF'
DB_HOST=localhost
DB_PORT=3306
DB_NAME=mht_edu
DB_USER=mht_edu
DB_PASSWORD=你的数据库密码
PORT=3002
NODE_ENV=production
JWT_SECRET=随机字符串32位以上
PROJECT_DOMAIN=https://你的域名.com
EOF

# 4. 启动后端（指定3002端口）
pm2 start dist/main.js --name mht-edu-api -- -p 3002

# 5. 保存PM2配置
pm2 save

# 6. 验证
curl http://localhost:3002/api/hello
```

## 🔧 Nginx配置（3002端口）

```nginx
server {
    listen 80;
    server_name 你的域名.com;
    
    root /www/wwwroot/mht-edu/dist-web;
    index index.html;
    
    # 前端路由
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API反向代理（注意端口改为3002）
    location /api/ {
        proxy_pass http://127.0.0.1:3002;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        client_max_body_size 50m;
    }
}
```
