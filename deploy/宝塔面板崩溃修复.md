# 宝塔面板 bt restart 后无法访问 - 紧急修复

## 🚨 紧急情况处理

### 情况一：SSH还能连接

```bash
# 1. 启动宝塔面板
/etc/init.d/bt start

# 2. 启动MySQL
/etc/init.d/mysqld start

# 3. 启动Nginx
/etc/init.d/nginx start

# 4. 查看面板信息
bt default
```

---

### 情况二：宝塔面板打不开

宝塔面板重启后需要等待1-2分钟。

**获取面板地址：**
```bash
bt default
```

会显示类似：
```
==================================================================
外网面板地址: http://你的IP:8888/abc123
内网面板地址: http://192.168.x.x:8888/abc123  
username: xxxxx
password: xxxxx
==================================================================
```

---

### 情况三：SSH也连不上

**阿里云控制台操作：**

1. 登录 https://ecs.console.aliyun.com
2. 找到你的服务器实例
3. 点击【重启】
4. 等待2-3分钟后重试SSH连接

---

### 情况四：宝塔面板彻底损坏

```bash
# 修复宝塔面板
curl http://download.bt.cn/install/update6.sh | bash

# 或者完全重装（不会删除网站数据）
# 先卸载
/etc/init.d/bt stop
rm -f /etc/init.d/bt
rm -rf /www/server/panel

# 再安装
yum install -y wget && wget -O install.sh http://download.bt.cn/install/install_6.0.sh && sh install.sh
```

---

## 🔍 排查命令

```bash
# 查看宝塔状态
/etc/init.d/bt status

# 查看端口
netstat -tlnp | grep 8888

# 查看进程
ps aux | grep bt

# 查看错误日志
cat /www/server/panel/logs/error.log | tail -50
```

---

## 📋 服务启动命令汇总

```bash
# 宝塔面板
/etc/init.d/bt start

# MySQL
/etc/init.d/mysqld start

# Nginx
/etc/init.d/nginx start

# PHP
/etc/init.d/php-fpm-74 start  # 根据版本

# Redis
/etc/init.d/redis start

# 全部启动
/etc/init.d/bt start && /etc/init.d/mysqld start && /etc/init.d/nginx start
```

---

## ⚠️ 重要提示

1. **不要频繁重启** - 等待服务完全启动
2. **检查内存** - `free -m` 确保有足够内存
3. **检查磁盘** - `df -h` 确保有空间
