# 网站访问问题完整排查清单

## 问题现象

curl 测试返回 200 OK，但浏览器无法正常访问网站。

---

## 排查步骤

### 1. 确认问题现象

**浏览器显示什么？**

- [ ] 无法访问此网站 / 连接超时
- [ ] 404 Not Found
- [ ] 502 Bad Gateway
- [ ] 503 Service Unavailable
- [ ] SSL 证书错误
- [ ] 下载文件而不是显示页面
- [ ] 空白页面
- [ ] 其他：______

---

### 2. 测试域名访问

```bash
# 测试 HTTP
curl -I http://你的域名/

# 测试 HTTPS
curl -I https://你的域名/

# 测试首页内容
curl http://你的域名/ | head -20

# 测试 PHP 文件
curl http://你的域名/index.php | head -20
```

---

### 3. 检查 DNS 解析

```bash
# Ping 域名
ping 你的域名

# 使用 dig 查询
dig 你的域名

# 检查是否解析到正确的服务器 IP
curl -I http://服务器IP/
curl -I -H "Host: 你的域名" http://服务器IP/
```

**对比：**
- 服务器实际 IP：`curl ifconfig.me`
- 域名解析 IP：`dig +short 你的域名`

---

### 4. 检查端口和服务

```bash
# 检查 Nginx 进程
ps aux | grep nginx

# 检查端口监听
netstat -tlnp | grep -E ':80|:443'
ss -tlnp | grep -E ':80|:443'

# 检查防火墙
firewall-cmd --list-all 2>/dev/null || iptables -L -n | head -20
```

---

### 5. 检查 SSL 证书

```bash
# 查看证书文件
ls -la /www/server/panel/vhost/cert/你的域名/

# 查看证书有效期
openssl x509 -in /www/server/panel/vhost/cert/你的域名/fullchain.pem -noout -dates

# 测试 HTTPS 连接
openssl s_client -connect 你的域名:443 -servername 你的域名 </dev/null 2>/dev/null | openssl x509 -noout -dates
```

**常见问题：**
- 证书过期
- 证书文件不存在
- 证书域名不匹配

---

### 6. 查看日志

```bash
# Nginx 错误日志
tail -50 /www/wwwlogs/nginx_error.log

# 网站错误日志
tail -50 /www/wwwlogs/你的域名.error.log

# 网站访问日志
tail -20 /www/wwwlogs/你的域名.log

# PHP-FPM 错误日志
tail -50 /www/server/php/74/var/log/php-fpm.log
```

---

### 7. 检查网站配置

```bash
# 查看配置文件
cat /www/server/panel/vhost/nginx/你的域名.conf

# 测试配置语法
/www/server/nginx/sbin/nginx -t

# 查看所有网站
ls -la /www/server/panel/vhost/nginx/
```

---

### 8. 对比正常网站

```bash
# 对比配置文件
diff /www/server/panel/vhost/nginx/正常域名.conf /www/server/panel/vhost/nginx/问题域名.conf

# 对比网站目录
ls -la /www/wwwroot/正常域名/
ls -la /www/wwwroot/问题域名/
```

## 常见问题解决

### 问题 1：DNS 解析错误

**症状：** ping 不通，curl 显示 "Could not resolve host"

**解决：**
- 检查域名是否已解析
- 检查 DNS 服务器设置
- 等待 DNS 生效（最多 48 小时）

---

### 问题 2：SSL 证书问题

**症状：** HTTPS 访问失败，证书错误

**解决：**
```bash
# 重新申请证书（宝塔面板）
# 网站 → SSL → Let's Encrypt → 申请

# 或使用自签名证书测试
openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
  -keyout /www/server/panel/vhost/cert/你的域名/privkey.pem \
  -out /www/server/panel/vhost/cert/你的域名/fullchain.pem \
  -subj "/CN=你的域名"
```

---

### 问题 3：伪静态规则问题

**症状：** 首页正常，内页 404

**解决：**
```bash
# 查看伪静态规则
cat /www/server/panel/vhost/rewrite/你的域名.conf

# 检查是否包含伪静态配置
grep "include.*rewrite" /www/server/panel/vhost/nginx/你的域名.conf
```

---

### 问题 4：PHP 解析问题

**症状：** 下载 PHP 文件或显示源码

**解决：**
```bash
# 确认 PHP 配置存在
ls -la /www/server/panel/vhost/nginx/enable-php-*.conf

# 检查网站配置是否引用
grep "enable-php" /www/server/panel/vhost/nginx/你的域名.conf

# 重启 PHP-FPM
/etc/init.d/php-fpm-74 restart
```

---

### 问题 5：权限问题

**症状：** 403 Forbidden

**解决：**
```bash
# 修复目录权限
chown -R www:www /www/wwwroot/你的域名/
chmod -R 755 /www/wwwroot/你的域名/

# 检查 .user.ini
cat /www/wwwroot/你的域名/.user.ini

# 如果需要修改
chattr -i /www/wwwroot/你的域名/.user.ini
```

---

### 问题 6：防火墙问题

**症状：** curl 本地成功，外部访问失败

**解决：**
```bash
# 检查防火墙
firewall-cmd --list-all

# 开放端口
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload

# 检查阿里云安全组
# 阿里云控制台 → 实例 → 安全组 → 配置规则
```

## 浏览器问题

### 清除缓存

1. Chrome: Ctrl + Shift + Delete
2. 或使用无痕模式：Ctrl + Shift + N

### 检查混合内容

如果 HTTP 能访问，HTTPS 不能：
- 检查页面是否有 HTTP 资源
- 浏览器控制台可能有 "Mixed Content" 错误

---

## 快速诊断命令

```bash
#!/bin/bash
DOMAIN="你的域名"

echo "=== 1. DNS 解析 ==="
dig +short $DOMAIN

echo -e "\n=== 2. HTTP 测试 ==="
curl -I http://$DOMAIN/

echo -e "\n=== 3. HTTPS 测试 ==="
curl -I https://$DOMAIN/

echo -e "\n=== 4. Nginx 状态 ==="
ps aux | grep nginx | grep -v grep

echo -e "\n=== 5. 端口监听 ==="
netstat -tlnp | grep -E ':80|:443'

echo -e "\n=== 6. 错误日志 ==="
tail -10 /www/wwwlogs/$DOMAIN.error.log

echo -e "\n=== 7. 配置测试 ==="
/www/server/nginx/sbin/nginx -t
```
