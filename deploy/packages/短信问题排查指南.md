# 短信功能问题排查指南

## 问题现象
短信发送失败或无响应

## 可能的原因

### 1. 阿里云SDK未安装
检查：
```bash
ls -la /www/wwwroot/mht-edu/server/node_modules/@alicloud/
```

如果不存在 `dysmsapi20170525` 目录，需要安装：
```bash
cd /www/wwwroot/mht-edu/server
npm install @alicloud/dysmsapi20170525 @alicloud/openapi-client --save
```

### 2. 数据库配置缺失或错误
检查数据库中的短信配置：
```sql
SELECT config_key, config_value FROM site_config WHERE config_key LIKE 'sms_%';
```

需要配置的字段：
| 字段 | 说明 | 示例 |
|------|------|------|
| sms_access_key_id | 阿里云AccessKeyId | LTAI5t... |
| sms_access_key_secret | 阿里云AccessKeySecret | xxx... |
| sms_sign_name | 短信签名 | 棉花糖教育 |
| sms_template_code | 短信模板ID | SMS_123456 |
| sms_enabled | 是否启用 | 1 |

更新配置：
```sql
UPDATE site_config SET config_value = '你的AccessKeyId' WHERE config_key = 'sms_access_key_id';
UPDATE site_config SET config_value = '你的AccessKeySecret' WHERE config_key = 'sms_access_key_secret';
UPDATE site_config SET config_value = '你的签名' WHERE config_key = 'sms_sign_name';
UPDATE site_config SET config_value = '你的模板ID' WHERE config_key = 'sms_template_code';
UPDATE site_config SET config_value = '1' WHERE config_key = 'sms_enabled';
```

### 3. 后端未重新构建
修改配置后需要重新构建：
```bash
cd /www/wwwroot/mht-edu/server
pnpm build
# 或
npm run build
```

### 4. 服务未重启
```bash
pm2 restart mht-edu-api
# 或
pm2 restart mht-edu-server
```

## 快速修复步骤（宝塔面板）

### 步骤1：安装SDK
打开终端执行：
```bash
cd /www/wwwroot/mht-edu/server
npm install @alicloud/dysmsapi20170525 @alicloud/openapi-client --save
```

### 步骤2：配置数据库
登录宝塔面板 → 数据库 → mht_edu → 管理 → phpMyAdmin

执行SQL：
```sql
-- 查看当前配置
SELECT * FROM site_config WHERE config_key LIKE 'sms_%';

-- 更新配置（替换为你的实际值）
UPDATE site_config SET config_value = 'LTAI5t...' WHERE config_key = 'sms_access_key_id';
UPDATE site_config SET config_value = 'xxx...' WHERE config_key = 'sms_access_key_secret';
UPDATE site_config SET config_value = '棉花糖教育' WHERE config_key = 'sms_sign_name';
UPDATE site_config SET config_value = 'SMS_123456' WHERE config_key = 'sms_template_code';
UPDATE site_config SET config_value = '1' WHERE config_key = 'sms_enabled';
```

### 步骤3：重新构建并重启
```bash
cd /www/wwwroot/mht-edu/server
pnpm build
pm2 restart mht-edu-api
```

### 步骤4：测试
查看日志：
```bash
pm2 logs mht-edu-api --lines 50
```

发送测试短信：
- 访问管理后台
- 系统配置 → 短信配置
- 点击"测试短信"

## 开发模式说明

如果 `sms_enabled` 为 0 或配置不完整，系统会进入开发模式：
- 发送短信时返回模拟成功
- 验证码默认为 `123456`

这在开发测试时很有用，但生产环境需要配置真实的短信服务。

## 阿里云短信配置说明

### 1. 获取AccessKey
登录阿里云控制台 → 右上角头像 → AccessKey管理

### 2. 开通短信服务
阿里云控制台 → 产品与服务 → 短信服务 → 开通

### 3. 申请签名
短信服务 → 国内消息 → 签名管理 → 添加签名

### 4. 申请模板
短信服务 → 国内消息 → 模板管理 → 添加模板

### 5. 模板内容示例
```
您的验证码为${code}，5分钟内有效，请勿泄露给他人。
```

## 常见错误

| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| isv.SMS_SIGNATURE_ILLEGAL | 签名不合法 | 检查签名是否已审核通过 |
| isv.TEMPLATE_MISSING_PARAMETERS | 模板缺少参数 | 检查模板参数是否正确 |
| isv.INVALID_JSON_PARAM | JSON参数错误 | 检查templateParam格式 |
| isv.MOBILE_NUMBER_ILLEGAL | 手机号格式错误 | 检查手机号格式 |
| isv.AMOUNT_NOT_ENOUGH | 余额不足 | 充值短信套餐 |
