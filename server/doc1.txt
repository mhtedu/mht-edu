棉花糖教育成长平台 - 完整开发计划

目标：快速构建一个基于 LBS 的家教信息撮合平台，整合个体教师、机构、家长，采用会员制 + 全员分销模式，不介入课时交易，仅提供信息展示、匹配沟通及教学辅助工具。存储：MySQL安装在 ECS 本地（自建数据库），所有数据落盘。技术栈：

后端：Node.js + NestJS（TypeScript）

服务器信息：Alibaba Cloud Linux 3.2104 LTS 64位

前端：电脑端、H5,、微信小程序 + 微信公众号 H5（uni-app）

管理后台：Vue3 + Element Plus

服务器：Alibaba Cloud Linux 3.2104 LTS（Docker 或直接部署）

推送：极光推送 / 微信订阅消息

核心原则：

1.平台不介入课时费交易，仅提供信息撮合和工具。

2.联系方式仅在订单匹配后对双方可见，不公开。

3.全员分销，两级佣金 + 城市代理提点，所有非课时消费均可设置独立分佣比例。

4.机构通过邀请链接/二维码挂靠教师，不强制派单，仅用于分佣关系。

5.所有端口均在前端个人中心里面切换。

二、功能模块划分（无时间周数，可并行开发）

模块一：基础架构与用户系统

NestJS 项目初始化，TypeScript，TypeORM，Redis 客户端，统一响应格式，JWT 认证。

手机端网页登录或者微信公众号、手机号绑定。（后面开发为微信小程序）

角色选择（家长/教师/机构，机构需提交资质审核）。

城市定位（前端获取经纬度，后端保存）。

会员套餐展示与购买（微信支付统一下单，回调更新用户会员状态）。

分销关系绑定（生成邀请链接/海报，注册时绑定邀请人，自动计算二级邀请人）。

模块二：LBS 需求发布与抢单

家长端发布需求表单（科目、课酬、年级、性别、地址）。

教师端 LBS 订单列表（基于距离排序，只显示付费教师的可抢订单）。

抢单逻辑（Redis 分布式锁，更新订单状态，创建匹配记录）。

订单详情页：双方解锁联系方式（仅匹配后可见，记录日志）。

订单状态流转：待抢单 → 已匹配 → 沟通中/试课中 → 已完成/已解除（解除后回流公海池）。

定时任务：自动解除超时未沟通的订单。

模块三：机构与教师挂靠

机构入驻申请（提交营业执照等），管理后台审核。

机构生成邀请链接/二维码（带 code）。

教师点击邀请链接扫码，确认挂靠，更新 affiliated_org_id。

机构后台查看挂靠教师列表、分佣记录。

当挂靠教师产生消费时，自动为机构创建分佣记录（level_type=4，比例从商品/会员套餐的 commission_org_rate 字段获取）。

模块四：全员分销与佣金结算

支付成功回调触发分佣：根据购买者的邀请链（一级、二级）、城市代理、挂靠机构，分别创建佣金记录。

佣金明细查询（用户端展示）。

提现申请（用户发起，管理后台审核，转账后更新状态）。

模块五：教师工具与商城

教师排课工具（日历视图，增删改查课时记录）。

教辅商城（商品列表、购买、支付触发分佣）。

管理后台商品管理（CRUD，设置分佣比例）。

模块六：首页与广告

城市定位显示，轮播广告（从数据库读取）。

学科筛选条，双列信息流（根据用户角色展示教师或订单，LBS 排序）。

中部横幅广告。



模块七：消息推送与定时任务

微信订阅消息/极光推送：订单被抢、新订单发布。

定时任务：自动解除匹配订单、清理过期订单、检查会员过期。

模块八：管理后台

Vue3 独立项目，使用 Element Plus。

功能：用户管理、订单管理、商品管理、会员套餐管理、广告位管理、机构审核、佣金管理、城市代理管理、数据统计。
