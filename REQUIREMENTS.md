# 棉花糖教育成长平台 - 完整需求文档

## 一、核心商业逻辑与角色定义

### 1.1 核心壁垒：双向付费墙 + 分销裂变

- **婚恋机制**：设置沟通门槛（付费解锁联系方式）
- **滴滴机制**：解决匹配效率（LBS排序 + 抢单池）

### 1.2 角色定义与关系映射

| 映射角色 | 平台角色 | 核心职责 | 盈利模式 |
|---------|---------|---------|---------|
| 女方 | 家长 | 发布需求、筛选教师、购买课时 | 消费方（支付会员费/课时费） |
| 男方 | 个体教师 | 接单、授课、发展下线 | 接单收入、分销佣金、会员费 |
| 红娘 | 机构老板 | 管理旗下教师、派单、对接家长 | 管理抽成、机构会员费 |
| 区域合伙人 | 城市代理商 | 地推、入驻审核、本地运营 | 辖区内流水分成、广告分成 |

### 1.3 核心撮合机制

1. **LBS排序**：基于GPS坐标，利用空间索引进行距离排序
2. **双向抢单池**：需求发布后进入"公海池"，付费教师/机构查看详情并抢单
3. **解约回流机制**：沟通/试课失败后，订单自动回流公海池

## 三、分销与锁客系统

### 3.1 锁客逻辑

用户点击分享链接/扫码 → 记录推荐人ID → 关系永久锁定

### 3.2 分佣类型

| 分佣类型 | 比例 | 说明 |
|---------|------|------|
| 会员费分佣 | 20% | A推荐B购买年费会员，A获得20%佣金 |
| 交易佣金分佣 | 1% | 教师完成订单，推荐人获得1%佣金 |
| 机构抽成 | 自定义 | 机构从旗下教师收入中抽成（如10%） |
| 城市代理分润 | 5% | 辖区内所有交易的分成 |

## 四、功能模块规划

### 4.1 首页与LBS模块

#### 功能点：
- [x] 城市定位（IP定位 + wx.getLocation）
- [ ] 双列混合流（模糊处理 vs 完整信息）
- [ ] 顶部Tab切换【找老师】/【找学生】
- [ ] 学科筛选
- [ ] 广告位（轮播图 + 信息流广告）
- [ ] 教具商城入口

### 4.2 需求发布与匹配流

#### 订单状态机：
```
0-待抢单 → 1-已接单沟通中 → 2-试课中 → 3-已签约 → 4-已完成
                                   ↓
                              5-回流公海池
```

#### 流程逻辑：
1. 家长发布需求（科目、课时费、地址、孩子情况）
2. 系统推送给5km内匹配的付费教师
3. 教师付费解锁联系方式并抢单
4. 首个抢单教师锁定订单
5. 超时(24h)自动释放订单

### 4.3 即时通讯

- 支持文字、语音、图片消息
- 仅付费会员可主动发起聊天
- 消息记录云端存储

### 4.4 支付系统

- 微信支付V3对接
- 支持会员购买、课时费预付
- 平台担保模式（资金监管）
